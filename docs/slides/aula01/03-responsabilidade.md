## Responsabilidade

Responsabilidade define a atribuição clara de deveres a uma unidade de software para manter consistência estrutural (Larman, 2004).

Princípio central:
> Comportamento deve residir próximo aos dados que ele protege (Martin, 2008).

Violação gera:
- classes anêmicas
- serviços Deus
- propagação de mudança
