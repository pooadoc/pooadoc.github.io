## Objetivos da aula

Ao final você deve conseguir:

- Ler código como **arquitetura**
- Identificar **complexidade acidental**
- Localizar **responsabilidades** e **fronteiras**
- Distinguir **domínio** vs **orquestração**
- Explicar **invariantes** e por que elas sustentam confiabilidade
- Preparar refatoração com **testes de caracterização**

## Premissas (o que não vamos revisar)

- Sintaxe OO básica (classes, getters/setters, herança “por herança”)
- Estruturas de dados (listas, mapas etc.)
- Conceitos de Web (HTTP, MVC)

!!! note "Professor"
    Deixe claro: “Revisão aqui é de **modelo mental**, não de sintaxe.”
